
# План: Перевод Аналитика в технический персонал

## Обзор

Роль **Аналитик** переводится в категорию технического персонала для соответствия стратегии интеграции Промпт-Инженера, где Аналитик выполняет функции внутреннего брифера, а не участника коллегиальных обсуждений.

---

## Изменения

### 1. Обновление конфигурации роли

**Файл:** `src/config/roles.ts`

```typescript
// Строка 113-120: изменить isTechnicalStaff
analyst: {
  icon: LineChart,
  color: 'text-hydra-analyst',
  label: 'role.analyst',
  description: 'staffRoles.description.analyst',
  isTechnicalStaff: true,  // <-- было false
  cardVariant: 'analyst',
},
```

---

## Что произойдёт автоматически

После изменения:

1. **Штатное расписание** (`/staff-roles`):
   - Аналитик переместится из группы «Эксперты» в группу «Технический персонал»
   - Рядом с названием появится бейдж с иконкой гаечного ключа (Wrench)

2. **Подготовка к Фазе 1** плана Промпт-Инженера:
   - При реализации фильтрации в селекторах моделей Аналитик будет автоматически исключён из коллегиальных чатов (вместе с Архивариусом, Промпт-Инженером и Логистиком)

---

## Примечание о чекбоксе

Чекбокс «Технический персонал» в панели деталей роли остаётся **заблокированным** (`disabled`). Это корректное поведение:
- Статус техперсонала — системный параметр, определённый в коде
- Пользователь не должен менять его вручную
- Чекбокс служит только для отображения текущего статуса

---

## Итоговый состав технического персонала

После изменения в категории техперсонала будут 4 роли:

| Роль | Функция |
|------|---------|
| **Архивариус** | Управление векторной памятью |
| **Аналитик** | Формирование ТЗ для ПИ |
| **Промпт-Инженер** | Оптимизация промптов |
| **Логистик** | Проектирование data-flow |
